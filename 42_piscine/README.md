# 42 Projects

*Public progress repository for the [42 Roma](https://42roma.it/) curriculum by [anachiossi](https://github.com/anachiossi).*

---

## About 42

42 is a tuition-free, peer-to-peer software engineering school with campuses worldwide. There are no teachers or lectures — learning happens through projects, peer evaluation, and self-direction. The curriculum starts with a 4-week intensive selection process called the **Piscine**, followed by a project-based common core.

---

## Projects

### [42_piscine](./42_piscine)

The Piscine is the entry program — 4 weeks of C and shell exercises designed to teach the fundamentals from scratch. This folder contains all exercises organized by topic rather than by the original module numbering.

| Folder | Contents |
|--------|----------|
| `output_functions/` | Character, string, and number output — `ft_putchar`, `ft_putnbr`, `ft_print_comb`, etc. |
| `strings_functions/` | String manipulation — copy, compare, search, case conversion, validation |
| `math_functions/` | Recursive and iterative math — factorial, power, fibonacci, primes, sqrt |
| `pointers_functions/` | Pointer manipulation — swap, div/mod, array reversal and sorting |
| `conversion_functions/` | Base and string conversion — `ft_atoi`, `ft_atoi_base`, `ft_convert_base` |
| `memory_functions/` | Heap allocation — `ft_strdup`, `ft_range`, `ft_strjoin`, `ft_split` |
| `commands/` | Unix command reimplementations — `ft_cat`, `ft_tail`, `ft_hexdump`, and more |
| `headers/` | Header-only exercises — macros and typedefs (`ft_abs`, `ft_boolean`) |
| `libft/` | The libft project — a small static library built from scratch |
| `rushes/` | Rush projects — 48-hour team exercises (rush00, rush01) |

## Notes

- All code follows the [42 Norm](https://github.com/42School/norminette) style guidelines.
- Test `main()` functions are either in a separate `main.c` or commented out inside the `.c` file, depending on how the exercise was originally written.
- AI was used as a support tool for organization, planning, and concept clarification — never to write or complete any function or algorithm.
